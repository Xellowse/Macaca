CODING STYLE GUIDE
==================

1. All Project is following google C/C++ convention.
   About the google convention, you can refer to
   http://google.github.io/styleguide/

2. Function Naming should follow the prefix order:
   [Ctor/Ut][Hal/Hw/Osal/Iif][Module Name][Function Name]

3. C File Naming should follow the prefix order:
   [mod]_[ctor/ut]_[hal/hw/osal/iif]_[Module Name].c

4. Header File Naming should follow the prefix order:
   [ctor]_[hal/hw/osal/iif]_[Module Name]_[pub].h

5. Binary(lib) Naming should follow the prefix order:
   a.  lib[ctor/ut]_[hal/hw/osal/iif]_[Module Name].so
   b.  [ctor/ut]_[hal/hw/osal/iif]_[Module Name].ko

6. All project version control is following semantic
   versioning. About the semantic versioning, you can
   refer to https://semver.org/

7. The version should hiden into the binary of the code
   with the string format:
   sv#[ctor/ut]_[hal/hw/osal/iif]_[Module Name]_ver:[version]

8. There are some rules for each part's version
   control. More information should be included in
   meta data of semantic versioning as following
   description.

   *IIF :

    Format:
      [Major].[Minor].[Patch]+[SubLayerl Major].[SubLayer2 Major]

    note1 : SubLayer's major number should included in IIF's
            meta data that user can identify which subLayers'
            version is match to this IIF. The order of SubLayer1
            and SubLayer2 in version field is up to developer.
            In this project, IIF's SubLayer1 is module HAL.
            SubLayer2 is module2 HAL.

    note2 : SubLayer's major update is the non-backward compatible
            API change, so IIF should be modified for new feature
            to match the new API. Therefore, IIF's minor should update.

          ex:
            1. Module  HAL: 0.0.0 update to 1.0.0
               Module2 HAL: 0.0.0
               IIF        : 0.0.0+0.0 update to 0.1.0+1.0

            2. Module  HAL: 1.0.0
               Module2 HAL: 0.0.0 update to  1.0.0
               IIF        : 0.1.0+1.0 update to 0.2.0+1.1

   *IIF Ctor:

    Format:
      [Major].[Minor].[Patch]+[Connection]

    note:  Connection is the description of what modules
           connect to IIF. Using some easy character to
           describe it.

           ex:
             0.0.0+mm2 : Module and module2 conncet to IIF.
             0.0.0+m   : Only module connect to IIF.
             0.0.0+m2  : Only module2 connect to IIF.

   *HAL/HW/OSAL Ctor:

    Format:
      [Major].[Minor].[Patch]+[SubLayer Major]

    note: SubLayer Major is the number of major number
          of module which generated by this Ctor.

           ex:
             HAL      : 0.0.0 update to 1.0.0
             Ctor HAL : 0.0.0+0 update to 0.1.0+1

   *HAL/HW/OSAL UT:

    Format:
      [Major].[Minor].[Patch]+[SubLayer Major]

    note: SubLayer Major is the number of major number
          of the module which tested by this UT.

           ex:
             HAL      : 0.0.0 update to 1.0.0
             HAL UT   : 0.0.0+0 update to 0.1.0+1
